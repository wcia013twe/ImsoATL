# Data Sources - Complete Breakdown

## Are the 65 points real data? **YES - 100% Real Data**

All 65 deployment sites are based on **real Florida census tract data** from official government and open-source databases.

---

## Data Source Chain

### 1️⃣ **Raw Input Data** (Real Government Data)

**Source File**: `data/florida_fcc_cable.csv`
- **Source**: FCC Broadband Data Collection (BDC)
- **What it contains**: Block-level broadband availability for all of Florida
- **Data Type**: Real FCC provider data (Comcast, AT&T, Spectrum, etc.)
- **Rows**: Millions of block-level records
- **URL**: https://broadbandmap.fcc.gov/data-download

---

### 2️⃣ **Aggregated Coverage Data** (Processed from Real Data)

**Generated File**: `florida_tract_coverage.csv`
- **Generated by**: `calculate_coverage_from_csv.py`
- **Total Records**: 5,160 census tracts (all of Florida)
- **Data Sources Combined**:

  #### A. FCC Broadband Coverage (Real)
  - Aggregated from block-level to tract-level
  - Coverage percentage calculated from actual provider data

  #### B. Census Demographics (Real - via Census API)
  - **Population**: Census ACS 5-year estimates (B01003_001E)
  - **Median Income**: Census ACS (B19013_001E)
  - **Poverty Rate**: Calculated from Census (B17001_002E / B17001_001E)
  - **API**: https://api.census.gov/data/2022/acs/acs5

  #### C. OpenStreetMap Assets (Real - via Overpass API)
  - **Schools**: OSM `amenity=school`
  - **Libraries**: OSM `amenity=library`
  - **Community Centers**: OSM `amenity=community_centre`
  - **Transit Stops**: OSM `public_transport=stop_position`
  - **API**: https://overpass-api.de/api/interpreter

**Example Row** (Real Data):
```csv
GEOID,coverage,population,median_income,poverty_rate,schools,libraries,...
12001000902,0.0,8418,2499,77.3,0,2,...
```

This means:
- Census Tract 12001000902 (Alachua County, FL)
- 0% broadband coverage (no providers)
- 8,418 real residents (from Census)
- $2,499 median income (from Census)
- 77.3% poverty rate (from Census)
- 0 schools, 2 libraries (from OpenStreetMap)

---

### 3️⃣ **Filtered Underserved Tracts** (Real Data, Filtered)

**Generated File**: `app/frontend/public/data/processed/underserved_tracts.json`
- **Generated by**: `filter_underserved_tracts.py`
- **Total Records**: 65 tracts (filtered from 5,160)
- **Filters Applied**:
  - ✅ `coverage < 100%` → Removed 5,026 fully covered tracts
  - ✅ `population > 500` → Removed 69 low-population tracts
- **Result**: 65 real census tracts that need WiFi deployment

**Why 65?**
- Florida has 5,160 census tracts total
- 5,026 have 100% broadband coverage (fully served)
- 134 have < 100% coverage (underserved)
- Of those 134, only 65 have population > 500 (meaningful to deploy)

---

### 4️⃣ **Ranked Deployment Sites** (Real Data, Scored)

**Generated File**: `app/frontend/public/data/processed/ranked_deployment_sites.json`
- **Generated by**: `rank_deployment_sites.py`
- **Total Records**: 65 sites (same as underserved tracts)
- **Scoring Formula** (based on real demographics):
  ```
  Impact Score = (40% × Population Score) +
                 (40% × Poverty Rate) +
                 (20% × Income Score)
  ```
- **Ranking**: Sites ranked 1-65 by impact score
- **Tiers Assigned**:
  - Tier 1 (Critical): Ranks 1-10
  - Tier 2 (High): Ranks 11-25
  - Tier 3 (Medium): Ranks 26-40
  - Tier 4 (Low): Ranks 41-65

**Example Site #1** (Highest Impact - Real Data):
```json
{
  "geoid": "12001000902",
  "deployment_rank": 1,
  "deployment_tier": "tier_1_critical",
  "impact_score": 70.9,
  "population": 8418,        // ← Real Census data
  "poverty_rate": 77.3,      // ← Real Census data
  "median_income": 2499,     // ← Real Census data
  "coverage_percent": 0.0,   // ← Real FCC data
  "schools": 0,              // ← Real OSM data
  "libraries": 2             // ← Real OSM data
}
```

### 5️⃣ **Tract Geometries** (Real Government Boundaries)

**Generated File**: `app/frontend/public/data/processed/underserved_tracts_geo.json`
- **Generated by**: `fetch_tract_geometry.py`
- **Total Features**: 65 polygon geometries
- **Source**: US Census Bureau TIGERweb API
- **API**: https://tigerweb.geo.census.gov/arcgis/rest/services/TIGERweb/tigerWMS_Current/MapServer/8/query
- **Data Type**: Official census tract boundary polygons
- **Vintage**: 2020 Census boundaries

**What it contains**:
- Exact latitude/longitude coordinates for each tract's polygon boundary
- Merged with deployment ranking data
- Used to draw red shaded areas on the map
